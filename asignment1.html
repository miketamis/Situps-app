The Basic Idea

The idea is simple, a smart watch app that tracks how many sit ups you do daily and motivates you towards completing your set goal.

Similar apps

after a bit of google I found several apps for both Android and iOS, but none for any smart watch. The Android and iOS apps include:

App 1: Sit Ups

Very basic application.

Initial Setup:

when the user starts the app its asks to select one of four goals: 100, 150, 200 and 250 pushups in a row. Then it requires the user todo as many push ups as they can in one set, this is so it can determine where the user is at. It also require if the user is male or female.

How goals are set:

the goals are set manual by the user when they start the application for the first time. Once you have completed the goal, it puts you in strength training mode where it just repeat the same group of sets each day and no longer pushes you improve yourself. It say that if you want you can reset the application and set a new goal but this means you lose your log :(

Daily use:

Each day when you go to use the application it presents you with todays workout. This consist of about 4 sets of pushups. once started it takes you through each set stating the number of situps you have to do for each set. in between each set it has a cool-down timer before the next set.

After completing all of the sets it asks how you found these sets so it can give you an appropriate number of reps next time. This is done by selected one of three options "It was too easy.", "It was just right." and "It was too hard."

Then the user is presented with a progress screen where they can see how close they are to achieve their goal.

Achievement System:

Very basic, you can get three different trophy badge that you can post to your facebook wall.

  1. Bronze: when your 1/3 of the way to your goal.
  2. Silver: when your 2/3 of the way to your goal.
  3. Gold: when you have achieved your goal.

Data:

this app collects how many reps you do in each workout and lets you view it as list.

App Plan

I'm going to use the workout schedule found on twohundredsitups.com

The following sketch show the rough app flow:

When you open the app for the very first time you are present with the "Welcome" screen. Once the user hits the middle button they are taken to the "age" selection menu.

The "age" selection screen gives you three options which correspond to the ages in twohundredsitups.com initial test. Clicking top button sets the age to "under 30", clicking middle button sets the age to "30 to 39" and clicking bottom button set the age to "40 plus". It then take you to the "initial amount" test.

The "initial amount" amount screen wants you todo as many push up as you can, and then enter it, top button increase the amount, bottom decrease the amount. Middle button sets the amount and takes you to the main "menu".

The main menu is where you can find everything, for a starter all we need is settings and start. When you select start it will take you to the sit up screen. to get back to the main menu from anywhere you can hold down the Back Button

The "sit ups" screen will indicate how many sit ups you must perform, the progress bar at the top will show where you are at, dots indicate sit ups and arrows indicate rests. once the user has finished all their sit ups they hit the middle button to move on, which takes them to the "rest" screen.

The "rest" screen has how many seconds left you have to rest for. It will also have a pie chart in the background to give some visual feedback. If the user decides they don't want to rest the can skip the rest by clicking the middle button.

Once you have completed all of your sets, you will be presented with the "done" screen, where you get three options, if you select "Too easy" you go up an exercise, if you select "Just right" you stay on the current exersize, if you chose "Too hard" you go down an exersize

Languages, Frameworks and IDE

What I'm going with and why

  • Html - because I am use to it and the alternatives don't add much value
  • TypeScript - because its javascript (which i'm use too) but smarter, it has types to prevent you from silly mistakes at the start, and with a decent IDE it will do intelligent autocomplete.
  • Sass - because it CSS that just better, variables, nesting, extending, etc...
  • yeoman - because it helps me get started really quickly with all the frameworks and languages I want.
  • Browerify - lets me use the same structor you would in a node.js project (which I love) but for a webapp
  • Gulp - build tool of choice, will handle all the compiling of typescript, sass, browserify, etc
  • Bower - dependency manager to get stuff for webapp e.g. JQuery
  • NPM - dependency manager for build tools and for some stuff used it webapp
  • jQuery - because its a must have
  • React - so I dont really have to manipulate the DOM
  • Jest - Unit Testing: because I really like Jasmine and this takes Jasmine to a whole another level
  • Browser Sync - Gives me live reloads and CSS injection as well other cool features
  • Atom with TypeScripter - Because I like Atom (and know all the keyboard shortcuts and workflow) and Typescripter makes working with typescript fun
  • Chrome Dev Tools - Because it the least frustrating and I know how to use it; also you can blackbox files (e.g. JQuery) so that when you step through code you only see your code and not JQuery, Firebug doesn't have this and I feel as if its a killer feature
  • Source Tree - awesome GUI for git

What I'm not going with and why

  • Eclipse - because it clunky, really clunky
  • Sublime - because its typescript plugin isnt as fully featured as Atoms.
  • Webstorm - Really liked the look of it but I went with Atom because thats what im use too and webstorm didn't look like it was much better
  • Brackets - used this for a while and loved it, but it was really buggy and therefore sometimes unusable
  • CoffeeScript - because I feel like its just javascript just a little simpler but not worth it when it means JS code you find online isn't instantly usable and it doesn't have types like typescript
  • Jasmine - because Jest has Jasmine and doesnt require me to use PhantomJS because it has a vitural dom
  • AngularJS - awesome when your application is super simple but gets really annoying todo some more complex stuff
  • Dart - because I didn't want to learn a new language, and most tools etc out there are made for JS
